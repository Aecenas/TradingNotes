# TradingNotes v1 回滚方案

## 触发条件
- 关键路径不可用（Trade 无法创建/保存）。
- 数据损坏或无法读取（数据库初始化失败率异常）。
- 崩溃率明显上升且影响核心流程。

## 回滚步骤
1. 暂停对外分发 v1.0.0 构建。
2. 回退到上一稳定构建（若无则发布 hotfix 版本 `1.0.1`，禁用新增入口）。
3. 保留用户本地数据库文件，禁止自动迁移逆向变更。
4. 输出事故说明，收集受影响设备信息与日志。

## 数据安全策略
- 禁止在回滚流程中删除用户本地数据库与附件目录。
- 若检测到 DB key 丢失，仅提示恢复策略，不执行破坏性重建。

## 修复与再发布
1. 在修复分支复现问题并补测试。
2. 通过 CI 与手工回归清单。
3. 发布修复版本并更新 release notes。

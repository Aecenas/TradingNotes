# 交易复盘记录 App - PRD v1

## 1. 产品概述

### 1.1 产品名称

交易复盘记录 App（Android&ios）

### 1.2 产品定位

一款面向主动交易者的结构化交易记录与复盘工具，用于替代 Excel 手工记录。  
产品核心目标不是“记流水账”，而是帮助用户以统一的数据结构记录交易的完整生命周期，支持后续复盘、检索、统计和策略优化。

### 1.3 目标用户

主动交易者，尤其是需要长期积累交易样本、复盘执行和优化策略的人，包括但不限于：

- 全球市场投资交易者

- 加密货币合约交易者

- 股票 / ETF / 期货 / 外汇 / 配对交易者

- 有明确交易体系，希望长期复盘的人

---

## 2. 产品目标

### 2.1 核心目标

本产品需要帮助用户完成以下事情：

1. 快速记录一笔交易的完整过程

2. 支持从简单单腿交易到复杂多腿 / 配对 / 分批交易的统一表达

3. 支持记录交易中的关键动作与市场事件

4. 支持交易结束后的结构化复盘

5. 为后续统计分析提供可用的数据基础

### 2.2 产品原则

1. **记录成本低**：尽量避免超长表单，优先时间线式记录

2. **结构统一**：简单交易和复杂交易使用同一套核心模型

3. **复盘优先**：重点不是记成交，而是形成“计划—执行—结果—复盘”的闭环

4. **可扩展**：后续可扩展统计、模板、导出、云同步等能力

---

## 3. 不做什么（MVP 边界）

### 3.1 本期不做

- 自动同步券商 / 交易所成交记录

- 实时行情接入

- 自动计算复杂期权 Greeks

- 自动画 K 线和图表分析

- 多端云同步

- 团队协作

- AI 自动生成复盘内容

### 3.2 本期只做

- 本地存储

- 手动录入

- 基础管理

- 交易时间线展示

- 基础筛选与统计

- 图片附件支持（先支持本地图片）

---

## 4. 核心概念与数据模型

产品采用三级结构：**Trade - Leg - Event**

---

### 4.1 Trade（交易项）

表示一笔完整交易主题，或一次完整交易战役。  
Trade 是复盘与统计的基本单位。

Trade 回答的问题是：

- 这笔交易在做什么

- 为什么做

- 属于什么策略和结构

- 最终结果如何

- 这笔交易有哪些结论和教训

#### Trade 主要字段

- `id`

- `title`：交易标题 / 主题

- `market`：市场（Crypto / US / HK / Futures / FX / Options / Other）

- `strategy_type`：策略类型

- `structure_type`：交易结构类型

- `thesis`：核心交易逻辑

- `entry_reason`：开仓理由

- `expectation`：预期

- `invalidation_condition`：失效条件

- `tags`：标签

- `status`：状态（Draft / Active / Closed / Archived）

- `start_time`

- `end_time`

- `summary_pnl`：汇总盈亏（可由系统计算或用户填写）

- `review_text`：自由复盘文本

- `review_score_execution`：执行评分

- `review_score_result`：结果评分

- `review_plan_followed`：是否按计划执行（Yes / Partial / No）

- `review_error_tags`：复盘标签

#### strategy_type 示例

- 趋势跟随

- 突破

- 左侧试错

- 均值回归

- 事件驱动

- 配对 / 相对价值

- 套利

- 其他

#### structure_type 示例

- 单腿方向性

- 多腿方向性

- 配对 / 相对价值

- 期权策略

- 套利 / 对冲

- 其他

---

### 4.2 Leg（腿）

表示 Trade 下的一个具体持仓腿。  
一个 Trade 可以有一个或多个 Leg。

Leg 用于表达：

- 单腿交易

- 分腿构成的多腿交易

- 配对交易中的多 A / 空 B

- 各种策略中的子持仓结构

#### Leg 主要字段

- `id`

- `trade_id`

- `symbol`：标的

- `name`：标的名称（可选）

- `direction`：方向（Long / Short）

- `instrument_type`：工具类型（Spot / Perp / Futures / Stock / ETF / Option / Other）

- `leverage`：杠杆（可选）

- `account`：账户 / 平台（可选）

- `currency`：计价货币（可选）

- `initial_plan_entry_price`：初始计划开仓价（可选）

- `initial_stop_loss`：初始止损位（可选）

- `initial_risk_budget`：初始分配风险额度（可选）

- `initial_atr`：初始 ATR / 波动参数（可选）

- `note`：备注

- `sort_order`

---

### 4.3 Event（事件）

表示交易过程中发生的关键动作、信息和判断变化。  
Event 分为两类：

#### A. Trade 级 Event

与整个 Trade 有关，不归属于具体 Leg。

例如：

- 关键市场事件

- thesis 更新

- 计划更新

- 交易环境变化

- 普通备注

#### B. Leg 级 Event

归属于某个 Leg，用于记录具体动作。

例如：

- 开仓

- 加仓

- 减仓

- 平仓

- 移动止损

- 移动目标位

- 对冲

- 换月 / 展期

- 普通备注

#### Event 通用字段

- `id`

- `trade_id`

- `scope_type`：`trade` 或 `leg`

- `leg_id`：当 scope_type=leg 时存在

- `event_type`

- `event_time`

- `title`：事件简标题

- `note`：自由备注

- `image_attachments`：图片附件列表

- `created_at`

- `updated_at`

#### Event 结构化字段（按类型动态使用）

- `price`

- `quantity`

- `notional`

- `risk_delta`

- `stop_loss_before`

- `stop_loss_after`

- `target_before`

- `target_after`

- `atr`

- `reason`

- `pnl_realized`

- `related_market_context`

> 说明：  
> Event 不能只有自由文字。  
> 必须同时支持“结构化字段 + 自由备注”，这样未来才能做统计和复盘分析。

#### Trade 级 event_type 示例

- `MARKET_EVENT`

- `THESIS_UPDATE`

- `PLAN_UPDATE`

- `GENERAL_NOTE`

#### Leg 级 event_type 示例

- `OPEN`

- `ADD`

- `REDUCE`

- `CLOSE`

- `MOVE_STOP`

- `MOVE_TARGET`

- `HEDGE`

- `ROLL`

- `GENERAL_NOTE`

## 5. 复盘模型

每个 Trade 在结束后需要支持复盘。  
复盘分为两部分：

### 5.1 结构化复盘

- 执行评分（1-5）

- 结果评分（1-5）

- 是否按计划执行（Yes / Partial / No）

- 主要错误标签（可多选）

- 是否值得纳入案例库（可选）

### 5.2 自由复盘文本

用户可填写：

- 本次交易最关键的认识

- 盈亏真正来源

- 主要问题

- 下次如何优化

- 其他心得

### 5.3 复盘标签示例

- 过早止盈

- 止损过紧

- 止损过松

- 加仓过猛

- 没按计划

- 逻辑对执行差

- 逻辑错误

- 拿得很好

- 教科书案例

- 情绪化交易

### 6.3 Trade 详情页

这是产品核心页面之一。

#### 页面内容建议

1. **顶部摘要区**
   
   - 标题
   
   - 市场
   
   - 策略
   
   - 状态
   
   - 标签
   
   - 时间范围
   
   - 汇总结果

2. **交易背景区**
   
   - 开仓理由
   
   - thesis
   
   - 预期
   
   - 失效条件

3. **Leg 列表区**
   
   - 展示所有 Leg
   
   - 可增删改

4. **事件时间线区**
   
   - 展示 Trade 下所有 Event（包括 trade 级和 leg 级）
   
   - 按时间排序
   
   - 在同一条时间线里清晰区分事件归属

5. **复盘区**
   
   - 若已结束，则显示复盘内容
   
   - 若进行中，则可先写草稿

#### 支持操作

- 编辑 Trade

- 添加 Leg

- 添加 Trade 级 Event

- 添加某个 Leg 的 Event

- 结束交易

- 归档交易

### 6.4 Leg 管理

#### 支持操作

- 新建 Leg

- 编辑 Leg

#### Leg 详情页内容

- 初始计划参数

### 6.5 Event 创建与编辑

Event 是用户高频使用的记录入口。

#### 6.5.1 创建 Trade 级 Event

用户在 Trade 下添加不归属某个 Leg 的事件。

适用于：

- 市场事件

- 交易判断变化

- 计划调整

- 普通笔记

#### 6.5.2 创建 Leg 级 Event

用户在某个 Leg 下添加具体动作。

适用于：

- 开仓

- 加仓

- 减仓

- 平仓

- 移动止损

- 普通备注

#### 6.5.3 Event 表单原则

- 根据 event_type 动态展示字段

- 同时允许自由备注

- 支持添加图片附件

- 时间默认当前时间，可手动修改

#### 6.5.4 Event 示例

**OPEN**

- 时间

- 价格

- 数量

- 名义市值

- 风险增量

- 备注

**MOVE_STOP**

- 时间

- 原止损位

- 新止损位

- 理由

- 备注

**MARKET_EVENT**

- 时间

- 标题

- 市场事件描述

- 对 thesis 的影响

- 图片附件

---

### 6.6 结束交易

用户应能明确执行“结束交易”操作。

#### 结束交易后

- Trade 状态变为 `Closed`

- 记录 end_time

- 允许填写最终复盘

- 允许补录最终平仓事件

- 汇总页可显示完整结果

#### 结束交易表单可包含

- 最终退出原因

- 是否按计划执行

- 执行评分

- 结果评分

- 复盘标签

- 自由复盘文本

---

### 6.7 归档

已结束的交易可以归档。

#### 归档目的

- 降低主列表噪音

- 便于历史案例管理

#### 归档要求

- 归档后仍可查看

- 默认不进入“进行中”视图

- 允许取消归档

---

### 6.8 基础统计页（MVP 简版）

MVP 可提供基础统计，而不追求复杂分析。

#### 建议展示

- 总交易数

- 盈利交易数 / 亏损交易数

- 不同市场的交易数量

> 复杂盈亏统计可以后续增强。

## 8. 交互原则

### 8.1 不做超级大表单

创建 Trade 时只填最关键的信息，剩余信息可后续补充。  
不要要求用户第一次就填完所有字段。

### 8.2 时间线优先

交易过程的记录以时间线为核心，而不是平铺型 Excel 表单。

### 8.3 表单动态化

Event 根据类型决定显示哪些字段，减少噪音。

### 8.4 记录速度优先

常用字段后续应支持：

- 默认值

- 快捷标签

- 最近输入复用

- 模板化（v2 再做）

## 10. 状态流转

### 10.1 Trade 状态

- `Draft`：草稿

- `Active`：进行中

- `Closed`：已结束

- `Archived`：已归档

### 10.2 建议流转

- 新建后可为 Draft 或 Active

- 用户开始正式记录后为 Active

- 完成交易后变为 Closed

- 历史整理后可设为 Archived

## 13. 验收标准（MVP）

满足以下条件即可视为 v1 可用：

1. 用户可以创建一个 Trade

2. 用户可以在 Trade 下创建一个或多个 Leg

3. 用户可以创建 Trade 级 Event 与 Leg 级 Event

4. 用户可以在 Trade 中看到完整时间线

5. 用户可以结束一笔交易并填写复盘

6. 用户可以查看交易列表并按基础维度筛选

7. 用户可以为事件添加图片

8. 数据本地持久化，重启 App 后不丢失

## 14. 典型使用场景

### 场景 1：单腿 BTC 趋势交易

- 创建 Trade：BTC 上破后趋势跟随

- 添加 1 个 Leg：BTCUSDT Perp，Long

- 记录 OPEN、ADD、MOVE_STOP、CLOSE

- 结束后填写复盘

### 场景 2：复杂分批建仓交易

- 创建 Trade：某波段左侧试错

- 添加 1 个 Leg

- 记录多次 OPEN / ADD / REDUCE / MOVE_STOP

- 记录一次 Trade 级事件：行业消息改变判断

- 最终结束并复盘

### 场景 3：配对交易

- 创建 Trade：多 A 空 B 的相对价值交易

- 添加 2 个 Leg：Long A / Short B

- 分别记录两个 Leg 的动作

- 记录 Trade 级的 thesis 更新

- 最终结束并做复盘

## 16. 一句话总结

这不是一个“交易记账本”，而是一个帮助交易者积累案例、复盘执行、优化体系的结构化交易复盘工具。  
v1 的重点是用统一的 Trade-Leg-Event 模型，可靠地承载简单和复杂交易，并让用户能够长期坚持记录。
